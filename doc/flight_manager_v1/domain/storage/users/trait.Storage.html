User Storage

Reservations Stored in User Storage

Since reservations are involved with money, and refunding is required, we need to record that "who did what reservations". Reservations are no longer stored as an independent state as in previous tries, since it makes little sense for a reservation to live with no owner, and we can easily get around the limitation by providing a void owner.

Nonetheless, the program still has some reservation storages. But they are mainly used to store temporary reservations (undone, unpaid etc).
